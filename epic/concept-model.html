Entities and Attributes

Profile

Users will be able to create a profile that allows them to like and upload murals.

One to one - One user can create one profile.

  • profileId (primary key)
  • profileActivationToken
  • profileEmail
  • profileHash
  • profileName
  • profileUsername

Mural

Murals will be submitted by users as well as by us developers.

Many to Many - Many murals can be submitted by many users

  • muralId (primary key)
  • muralLikeProfileId (foreign key 1)
  • muralProfileId (foreign key 2)
  • muralContent
  • muralDate
  • muralImage

Main Map

This is the entity for the large overview map with its page.

One to Many - One main map will have many drop points

  • mainMapId (primary key)
  • mainMapDropPointId (foreign key 1)
  • mainMapMuralId (foreign key 2)
  • mainMapContent

Route Maps

This is the entity for the routes page where pre-mapped routes are created based on walk time and proximity between murals. Routes are designed to ease the user's burden of planning and allows them to choose a predefined route to explore.

Many to Many - There will be many routes with many drop points.

  • routeId (primary key)
  • routeDropPointId (foreign key 1)
  • routeMuralId (foreign key 2)
  • routeMapContent

Drop Points

This is the entity for the points on the map indicating the location of a mural.

Many to Many - While there will be one drop point per mural, there will be many drop points on many maps.

  • dropPointId (primary key)
  • dropPointMuralId (foreign key 1)
  • dropPointProfileId (foreign key 2)
  • dropPointLocation

Like

This is the entity for the likes, which will be available on every mural.

Many to Many - There can be many likes on many murals

  • likeProfileID (foreign key 1)
  • likeMuralId (foreign key 2)
