Entities and Attributes

Profile

Users will be able to create a profile that allows them to like and upload murals.

One to many - One user can create one profile and can like many murals with their profile.

  • profileId (primary key)
  • profileActivationToken
  • profileEmail
  • profileHash
  • profileName
  • profileContent

Mural

Murals will appear on their own page and all have drop points on the main map, with some also having drop points on the route maps, as well.

One to Many - There are many murals and many maps, and each mural will have one drop point. (many side)

  • muralId (primary key)
  • muralRouteMapId (foreign key)
  • muralArtist
  • muralAddress
  • muralContent (pending based on wireframe)
  • muralCreationDate
  • muralImageUrl
  • muralLat
  • muralLong
  • muralTitle

Route Maps

This is the entity for the routes page where pre-mapped routes are created based on walk time and proximity between murals. Routes are designed to ease the user's burden of planning and allows them to choose a predefined route to explore.

Many to Many - There will be many routes with many drop points.

  • routeMapId (primary key)
  • routeMapImageURL
  • routeMapName
  • routeNeighborhoodLat
  • routeNeighborhoodLong
  • routeMapContent

Like

This is the entity for the likes, which will be available on every mural.

Many to Many - There can be many likes on many murals

  • likeProfileID (foreign key 1)
  • likeMuralId (foreign key 2)
